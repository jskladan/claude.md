# Git Practices Guidelines

Comprehensive Git workflow patterns for AI-assisted iterative development.

## Workflow
- **Branch strategy**: Feature branches only - never develop on main
- **Pre-commit**: Check current branch before committing significant changes
- **Co-authorship**: Always add `Co-authored-by: Claude Code <noreply@anthropic.com>`
- **CRITICAL: NO "Generated by" text**: NEVER add "Generated with [Claude Code]" or similar attribution text. Co-authorship is sufficient attribution.

## Development Phases

### Iterative Development Phase
- **Commit frequently**: Save work-in-progress, incomplete features, experimental approaches
- **Verbose commit messages**: Detailed descriptions including reasoning, blockers, next steps
- **Document decisions**: Capture architectural choices, failed attempts, lessons learned
- **Progressive refinement**: Build, test, refactor in small increments

### History Cleanup Phase
- **Strategic squashing**: Group related commits into logical, isolated changes
- **Concise final messages**: Clear, actionable summaries focusing on "what" not "how"
- **Preserve checkpoints**: Maintain significant milestones as separate commits
- **Clean narrative**: Create readable history that tells the story of the feature

## Commit Messages

### Style Decision Matrix

| User Request               | Trigger Words                                 | Style        | Format                  |
|----------------------------|-----------------------------------------------|--------------|-------------------------|
| **Explicit style**         | `/commit final`, `/commit dev`                | As requested | Follow style rules      |
| **Final indicators**       | "final", "ready", "complete", "ship", "prod"  | Concise      | Brief + bullet points   |
| **Development indicators** | "WIP", "checkpoint", "experiment", "test"     | Verbose      | Detailed context        |
| **Auto-detection**         | Context analysis                              | Best fit     | Based on change scope   |

### Development Phase Messages
- **Detailed and contextual**: Include reasoning, blockers, experiments
- **Document progress**: "WIP: implement user auth - OAuth integration complete, need session handling"
- **Capture decisions**: "Refactor: switch from Redis to SQLite - simpler deployment, sufficient for scale"
- **Note blockers**: "Fix: authentication flow - still debugging token refresh logic"

### Final Phase Messages
- **Concise and clear**: Focus on completed functionality
- **80 character subject line maximum**
- **What, not how**: Describe the change, not implementation details
- **Add context for breaking changes**: Include migration notes when necessary

### Commit Message Examples

**Good:**
```
# Simple changes
Fix toggle slider colors
Update virtual environment guidelines
Add missing import statements

# Complex changes with context
Replace HTTP server with Flask implementation

Breaks compatibility with direct file serving. Flask provides better
request handling and logging integration.

Migrate authentication to OAuth2

Replaces custom auth system. Requires new environment variables:
OAUTH_CLIENT_ID and OAUTH_CLIENT_SECRET.
```

**Bad (over-detailed for simple changes):**
```
Fix toggle slider colors
- Change background from bg-gray-600 to bg-dark-tertiary
- Update circle color from after:bg-white to after:bg-text-primary
- Apply Everforest theme colors consistently across all states
```

## Git Operations
- **Explanation**: Explain operations briefly, wait for approval before executing
- **Flags**: Prefer non-interactive flags (`--no-edit`, `--no-ff`)
- **Batching**: Batch related operations
- **CRITICAL: Attribution**: Use co-authorship ONLY. NEVER add "Generated with [Claude Code]" or any "Generated by" text.

### Rewording Non-Last Commit Messages

- **Check for alias**: Verify if user has `git reword` alias (used as `git reword SHA MESSAGE`):

```bash
reword = "!f() {\n  GIT_SEQUENCE_EDITOR=\"sed -i '' 1s/^pick/reword/\" GIT_EDITOR=\"printf \\\"%s\\n\\\" \\\"$2\\\" >\" git rebase -i \"$1^\";\n}; f"
```

- **Fallback method**: If no alias exists, propose setting it up or use manual method:

```bash
# Working method for non-interactive commit message rewrite
GIT_SEQUENCE_EDITOR="sed -i '' 1s/^pick/reword/" GIT_EDITOR="printf \"%s\n\" \"new commit message here\" >" git rebase -i <commit-hash>^
```

### Example: Careful Commit Analysis
```bash
# WRONG: Assuming all recent commits are related
git log --oneline -5
# c4d2ea8 Change theme to Everforest
# 4ba2b36 .
# 4f9eb5d Add Claude AI Assistant development guidelines
# d2254ba Previous unrelated commit

# RIGHT: Examine each commit individually
git show --name-only c4d2ea8  # Shows: template files (theme changes)
git show --name-only 4ba2b36  # Shows: CLAUDE.md (documentation)
git show --name-only 4f9eb5d  # Shows: CLAUDE.md, .cursor/rules (documentation)

# CONCLUSION: Only 4ba2b36 and 4f9eb5d are related (both CLAUDE.md)
# Theme commit (c4d2ea8) should remain separate
```

## Bug Fixes
- **Scope**: Minimal changes - limit scope to necessary fixes
- **Separation**: Keep bug fixes separate from features
